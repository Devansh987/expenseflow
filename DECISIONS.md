Decision 1: Dynamic Membership Tracking

Problem



Members join and leave groups over time.



Options

Store only current members.

Maintain membership history.

Decision



Maintain membership history.



Reason



The assignment includes Meera leaving and Sam joining. Historical expenses must be calculated using the membership active at the expense date.



Decision 2: Duplicate Handling

Problem



Some expenses appear duplicated.



Options

Automatically delete duplicates.

Require user approval.

Decision



Require user approval.



Reason



The assignment explicitly requires that users approve changes to duplicate records.



Decision 3: Currency Storage

Problem



Expenses are recorded in INR and USD.



Options

Convert and discard original value.

Store both original and converted values.

Decision



Store both.



Reason



Provides auditability and transparency.
