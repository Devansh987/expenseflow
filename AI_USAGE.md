# AI Usage Log

This document tracks how AI was utilized as a primary development collaborator to build ExpenseFlow.

### AI Tool Used
- DeepMind Agentic Coding Assistant (Antigravity)
- **Role:** Pair-programming collaborator.

### Development Strategy & Prompts
To avoid the "single bulk commit" red flag, I directed the AI using an incremental, phase-by-phase strategy. 

**Key System Prompts Used:**
> "Do NOT generate the entire project at once. Work incrementally, phase by phase. After every significant milestone, stop and tell me what was completed. Suggest a meaningful Git commit message. Keep the code clean, modular, and interview-ready."

### 3 Cases Where AI Produced Errors & How I Fixed Them

**1. Async SQLAlchemy & Alembic Configuration Error**
- **The Issue:** The AI set up the `database.py` to use `asyncpg`, but when initializing Alembic for database migrations, the autogenerated `env.py` was trying to run synchronous queries against the async engine, causing an `AttributeError` during migrations.
- **The Fix:** I reviewed the stack trace and directed the AI to update `alembic/env.py` to use `run_migrations_online()` with an `asyncio.run()` wrapper, allowing Alembic to properly await the PostgreSQL connection.

**2. Duplicate Imports in Router Initialization**
- **The Issue:** While registering new routers (like `expense`, `balance`, etc.) into `app/routers/__init__.py`, the AI used a regex find-and-replace tool with a target context that was too generic. This caused it to accidentally duplicate the `from app.routers import auth` line multiple times.
- **The Fix:** I caught the duplication while reviewing the file diffs outputted by the AI. I instructed the AI to run a specific replacement to clean up the duplicates and export the `__all__` array correctly.

**3. Pydantic `EmailStr` Missing Dependency**
- **The Issue:** The AI built the `UserBase` schema using `EmailStr` from Pydantic. However, when I started the server with `uvicorn app.main:app --reload`, the server crashed with an `ImportError: email-validator is not installed`. 
- **The Fix:** I pasted the exact stack trace back to the AI. The AI recognized that Pydantic requires the optional `[email]` extra or the `email-validator` package directly. The AI updated `requirements.txt` and executed the `pip install` command to resolve the crash.
